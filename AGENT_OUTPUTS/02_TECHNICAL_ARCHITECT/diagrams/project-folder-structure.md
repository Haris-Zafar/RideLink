## File Naming Conventions

### React Components

- **PascalCase**: `RideCard.jsx`, `UserProfile.jsx`
- **Folder per component** (if complex):
  ```
  RideCard/
  ├── index.jsx
  ├── RideCard.module.css
  └── RideCard.test.jsx
  ```

### Backend Files

- **camelCase**: `authController.js`, `userService.js`
- **Models**: `User.js`, `Ride.js` (PascalCase, singular)
- **Routes**: `auth.js`, `rides.js` (camelCase, plural)

### Utilities & Helpers

- **camelCase**: `formatters.js`, `validators.js`
- **Constants**: `constants.js` (all UPPERCASE inside)

## Import/Export Patterns

### Frontend (ES Modules)

```javascript
// Named exports (preferred for utilities)
export const formatDate = (date) => { ... };
export const formatCurrency = (amount) => { ... };

// Default exports (for components)
export default function RideCard({ ride }) { ... }

// Import
import RideCard from '@/components/rides/RideCard';
import { formatDate, formatCurrency } from '@/utils/formatters';
```

### Backend (CommonJS or ES Modules)

```javascript
// ES Modules (package.json has "type": "module")
export const authService = { ... };
export default authService;

// Import
import authService from './services/authService.js';
import { sendEmail } from './services/emailService.js';
```

## Code Organization Best Practices

### 1. Single Responsibility

Each file should have one primary purpose:

- ✅ `authService.js` handles authentication logic
- ❌ Don't mix authentication + user profile logic in one file

### 2. Colocation

Keep related files close:

```
components/
└── rides/
    ├── RideCard.jsx
    ├── RideCard.test.jsx
    └── RideCard.module.css
```

### 3. Absolute Imports (Frontend)

Use path aliases in `vite.config.js`:

```javascript
resolve: {
  alias: {
    '@': '/src',
    '@components': '/src/components',
    '@utils': '/src/utils',
  }
}
```

Then import:

```javascript
import RideCard from '@components/rides/RideCard';
import { formatDate } from '@utils/formatters';
```

### 4. Index Files for Clean Exports

```javascript
// components/rides/index.js
export { default as RideCard } from './RideCard';
export { default as RideList } from './RideList';
export { default as RideForm } from './RideForm';

// Import all at once
import { RideCard, RideList, RideForm } from '@components/rides';
```
